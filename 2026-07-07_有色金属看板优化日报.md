## 一、Git 部署可靠性（P0）

### 问题
- 17:00 cron 自动推送 GitHub Pages 时，因网络环境无代理导致 `git push` 必失败
- `login_and_save_cookies.py` 存在 `ImportError`，直接阻塞登录流程
- 多个脚本各自实现 git 操作，冲突处理逻辑不一致

### 改动
| 文件 | 改动 |
|------|------|
| `git_helper.py`（新增） | 统一封装：代理自动检测（7890/1087/7897/8888/6152）+ rebase 冲突自动 abort + 统一 timeout |
| `fill_and_verify.py` | 后置发布改用 `publish_to_github()`，解决 cron push 无代理必失败 |
| `sync_from_web.py` | 拉取改用 `git_pull_rebase()` |
| `excel_to_web.py` | 推送改用 `publish_to_github()` |
| `login_and_save_cookies.py` | 修复 `ImportError`：`from credentials import SMM_ACCOUNT` → `require_smm()` 延迟调用 |

### 效果
- 代理检测：先试直连 5s → 不通再逐端口探测，自动 fallback
- 冲突处理：rebase 冲突时自动 abort，不让仓库卡死
- 今晚 17:00 cron 不再因网络问题失败

### 问题
原看板所有卡片样式一致，日数据与月数据混在一起，没有视觉层次。

### 改动

**三色分区系统**：
| 分区 | 颜色 | 内容 |
|------|------|------|
| 日维度 | 蓝色（#2563eb） | 日价格明细表 + 4 张日趋势图 |

- 每张卡片标题后加彩色 badge（日趋势/月度/跨月/区间/预测/年度）

### 效果

## 四、在线编辑器优化

### 安全加固
| 项 | 旧 | 新 |
|----|----|----|
| Token 存储 | `localStorage`（永久，XSS 风险） | `sessionStorage`（关闭浏览器自动清除） |
| Token 验证 | 仅验证 `/user` | 额外验证 `/repos/...` 仓库权限 |
| API Header | `token` 前缀 | `Bearer` 前缀（GitHub 推荐） |

### 体验优化
- **409 冲突友好提示**：文件被其他人修改时提示"请刷新页面后重试"
- **草稿自动保存**：每 30 秒存 sessionStorage，关闭浏览器不丢编辑内容
- **返回看板链接**：工具栏顶部加"← 返回看板"
- **去掉用户名显示**：按需求恢复显示 `·`

## 六、预测模型升级

### 旧模型问题
普通 OLS 线性回归，等权重，无动量修正，对大宗商品价格预测效果差。

### 新模型
**双因子混合模型**：

```
预测值 = 加权趋势外推 × 70% + 动量修正 × 30%
```

| 组件 | 实现 |
|------|------|
| 加权线性回归 | 指数衰减权重（近月权重 1.0，6 月前权重 0.12） |
| 动量修正 | 最近 2 月变动率，捕获短期加速/减速 |
| EWMA 平滑 | alpha=0.4，替代简单 3 月 MA |
| 自适应置信区间 | 残差 SE × 1.15（预测月份不确定性增量） |

### 摘要增强
- 趋势方向（上涨/下跌/持平）
- 动量状态（加速/减速/平稳）
- 较本月涨跌百分比 + 箭头
- **免责声明**（紫色小字）：

> ⚠ 本预测仅基于历史价格数据的统计模型，未考虑宏观政策、供需突变、地缘事件等外部因素，结果仅供参考，不构成投资建议。

## 七、Bug 修复记录

### Bug 1：KPI 渲染崩溃
- **现象**：页面打开全空白
- **根因**：改 KPI 时漏掉 `cuAvg/alAvg/adcAvg` 变量声明
- **修复**：补回三个变量声明
- **提交**：`8e68689`

### Bug 2：数值精度丢失
- **现象**：WTI 原油显示 69 而非 69.38
- **根因**：`Math.round(sum/cnt)` 四舍五入丢失小数
- **修复**：`Math.round(sum/cnt * 100) / 100` 保留 2 位小数
- **提交**：`2b16624`

### Bug 3：大面积缺数（P0）
- **现象**：7/1、7/3 等日期 ADC12/A380/铝合金/钨粉/WTI 显示"—"
- **根因**：误把 `EXCEL_PATH` 从 `2026年有色金属市场价格.xlsx` 改为 `共享(2).xlsx`，后者是不完整副本
- **修复**：所有脚本路径改回 `2026年有色金属市场价格.xlsx`，重新导出 data.json
- **提交**：`e80d2b1`
- **教训**：不要随意更改数据源路径，改前必须 diff 验证

### Bug 4：误删 `daily_update_all.py`（P0，7/8 审计发现）
- **现象**：`daily_check_missed.py` 第 41/57 行调用 `daily_update_all.py`，但该文件已被删除
- **根因**：7/7 18:42 的 `f40bf8b` 误判为死代码并删除；17:00 自动化实际依赖它做缺数补抓
- **后果**：若今晚 17:00 有品种缺数，补抓会报 `FileNotFoundError`，链路断裂
- **修复**：从 git 历史恢复 `daily_update_all.py`，并将其 `EXCEL_PATH` 同步修正为 `2026年有色金属市场价格.xlsx`
- **提交**：`10fb93d`
- **教训**：删除文件前必须 grep 全仓库调用链，不能只看 cron/launchd 配置
